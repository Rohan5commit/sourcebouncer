# SourceBouncer Architecture

## System Overview

SourceBouncer is a Next.js 15 application that serves as a callable verification agent in the CROO agent economy. It accepts paid verification requests through CAP, analyzes claims using NVIDIA NIM, and returns structured trust reports.

## Task Lifecycle

1. **Intake** — Claims, sources, and context submitted via API or UI
2. **Validation** — Zod schemas validate all input fields
3. **CAP Invocation** — Payment initiated through CROO Agent Protocol
4. **Escrow** — USDC locked in smart contract
5. **Verification** — NVIDIA NIM analyzes each claim
6. **Report Generation** — Structured TrustReport produced
7. **Settlement** — Payment released on-chain
8. **Delivery** — Report returned to requester

## Verification Flow

- Each claim analyzed independently against provided sources
- AI produces verdict, confidence, evidence strength, and reasoning
- Aggregate scores calculated deterministically
- Summary generated by AI
- Malformed AI output falls back to safe unverifiable state

## CAP/ Payment Flow

- Provider registers service with pricing tiers
- Requester discovers agent on Store
- Negotiation via WebSocket
- USDC escrowed on Base
- Delivery proof submitted
- Payment released automatically

## Downstream Agent Reuse Flow

- Agent discovers SourceBouncer on Store
- Hires via CAP
- Receives structured JSON report
- Parses verdicts and scores
- Takes action: proceed, revise, reject, or flag
